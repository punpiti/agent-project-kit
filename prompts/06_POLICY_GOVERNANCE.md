# 06 — Policy / Governance / Curriculum Prompt

ใช้กับเอกสารหลักสูตร นโยบาย เสนอผู้บริหาร สภา คณะกรรมการ governance หรือ stakeholder-heavy document

```text
งานนี้ใช้ Spec–Eval–Loop Workflow
เน้น policy / governance / curriculum / executive communication

ก่อนเขียน ให้ระบุ:

1. Audience and Stakeholders
   - ใครคือผู้อ่านหลัก
   - ใครมีอำนาจตัดสินใจ
   - ใครอาจคัดค้าน
   - ใครได้รับผลกระทบ

2. Objective
   ต้องการ:
   - ขออนุมัติ
   - ขอทรัพยากร
   - อธิบายแนวทาง
   - ป้องกันข้อโต้แย้ง
   - สร้าง consensus
   - สื่อสารต่อสาธารณะ

3. Loop Diagnosis
   L1 = AI ช่วยร่าง/จัดโครง/หาช่องโหว่
   L2 = ต้องใช้ judgment ของผมเรื่อง politics, wording, feasibility, institutional fit
   L3 = ต้องใช้ feedback จากผู้บริหาร คณะกรรมการ stakeholder หรือข้อมูลจริง

4. Risk Check
   ตรวจ:
   - wording ที่เปิดช่องให้ตีความผิด
   - claim ที่แรงเกินหลักฐาน
   - commitment ที่ทำจริงไม่ได้
   - budget/manpower/time ที่ไม่ realistic
   - stakeholder ที่อาจรู้สึกเสียประโยชน์
   - ประเด็น governance ที่อาจถูกถาม

5. Draft
   เขียนให้:
   - ชัด
   - defend ได้
   - ไม่อวดเกินหลักฐาน
   - แยก vision กับ implementation
   - ระบุ decision points

6. Review Gate
   ระบุว่าอะไรต้องให้ผมตัดสินใจ
   อะไรควรเอาไปถาม stakeholder จริง
   อะไรต้องใช้ข้อมูลสนับสนุนเพิ่ม
```
