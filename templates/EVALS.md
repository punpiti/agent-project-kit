# EVALS / ACCEPTANCE CRITERIA

## Functional Criteria

- [ ] Output ทำงานตามโจทย์
- [ ] ครอบคลุม input สำคัญ
- [ ] handle edge cases ที่ระบุ
- [ ] ไม่มี regression ที่รู้จัก

## Quality Criteria

- [ ] อ่านเข้าใจง่าย
- [ ] structure ชัด
- [ ] ไม่ซับซ้อนเกินจำเป็น
- [ ] maintainable
- [ ] ไม่มี magic number / hardcoded path ที่ไม่อธิบาย

## Evidence Criteria

- [ ] มี test / check / citation / data / example รองรับ
- [ ] claim ไม่เกิน evidence
- [ ] ระบุสิ่งที่ยังไม่ได้ตรวจ

## Human Context Criteria

- [ ] ตรงกับ audience
- [ ] tone เหมาะ
- [ ] stakeholder risk ถูกพิจารณา
- [ ] decision point ชัด

## External Feedback Criteria

- [ ] ระบุว่า feedback ต้องเก็บจากใคร
- [ ] ระบุวิธีเก็บ feedback
- [ ] ระบุ decision rule
- [ ] มีแผนเอา feedback กลับมา update spec
